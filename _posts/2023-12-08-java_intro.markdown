# Java란 무엇인가?

**Java(자바)**는 객체 지향 프로그래밍 언어로, 다양한 운영체제에서 실행 가능한 **플랫폼 독립성**을 특징으로 합니다.  
1995년 Sun Microsystems(현재 Oracle)에 의해 처음 발표된 이후, 웹, 서버, 모바일, 임베디드 등 수많은 분야에서 활용되고 있습니다.

---

## Java의 핵심 특징

### ☑️ 플랫폼 독립성
- 자바는 **JVM(Java Virtual Machine)** 위에서 동작하여, 운영체제에 관계없이 프로그램 실행이 가능합니다.  
> “한 번 작성하면 어디서나 실행된다 (Write Once, Run Anywhere)”

### ☑️ 객체 지향 프로그래밍(OOP)
- 클래스와 객체 중심 구조로 **캡슐화, 상속, 다형성**을 활용하여 유지보수성과 재사용성이 뛰어납니다.

### ☑️ 자동 메모리 관리 (Garbage Collection)
- 개발자가 메모리를 직접 해제하지 않아도 되며, JVM이 자동으로 처리합니다.

### ☑️ 풍부한 표준 라이브러리
- 파일 I/O, 네트워크, DB, UI 등 다양한 기능이 **java.* 패키지**를 통해 제공됩니다.

### ☑️ 멀티스레딩 지원
- 스레드(Thread)를 사용하여 병렬 처리 및 백그라운드 작업이 가능합니다.

### ☑️ 보안성
- JVM에서 동작하며 샌드박스 구조로 외부 코드 실행 시에도 **보안이 강화**됩니다.

---

## Java 실행 구조

> ![자바 구조](/public/img/java/자바구조.png)

- **JDK (Java Development Kit)**  
  자바 개발에 필요한 모든 툴을 포함. (컴파일러, 디버거 등)

- **JRE (Java Runtime Environment)**  
  자바 애플리케이션을 **실행**하는 데 필요한 환경 (JVM 포함)

- **JVM (Java Virtual Machine)**  
  바이트코드(.class)를 실행하는 가상 머신으로, 플랫폼 독립성의 핵심 요소

- **API (Application Programming Interface)**  
  자바 프로그램에서 활용 가능한 다양한 라이브러리

🔗 [Java 8 SE 공식 문서 보기](https://docs.oracle.com/javase/8/docs/)

## Java의 장점

✅ **플랫폼 독립성**  
✅ **객체 지향 구조로 인한 유지보수성**  
✅ **풍부한 오픈소스와 생태계**  
✅ **강력한 보안성과 예외처리**  
✅ **강력한 멀티스레딩과 네트워크 기능**  
✅ **커뮤니티 및 학습 자료 풍부**

---

## Java의 단점

⚠️ **성능 이슈**  
- JVM 위에서 실행되기 때문에 네이티브 언어(C/C++)보다 속도가 느릴 수 있음

⚠️ **메모리 사용량**  
- 가비지 컬렉션으로 메모리 최적화에 한계가 있음

⚠️ **Verbose한 문법**  
- 코드가 장황해지는 경향이 있으며, 동일 기능 구현에 코드량이 많을 수 있음

⚠️ **모바일 개발의 제한성**  
- 안드로이드에서는 Kotlin이 대세로 떠오르며, 자바는 상대적으로 덜 쓰임

## 마무리

Java는 **오랜 역사와 강력한 생태계**를 가진 언어로, **대규모 시스템, 웹 애플리케이션, 백엔드 서버** 등에 적합합니다.  
자바의 철학과 구조를 이해하면 다른 언어를 학습할 때도 큰 도움이 됩니다.
